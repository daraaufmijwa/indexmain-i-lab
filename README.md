# indexmain-i-lab

本仓库用于归档和发布多个独立 HTML 页面。这些页面各自独立运行，不针对任何特定域名或网站。

## 项目简介

这是一个轻量级的 HTML 页面集合仓库。每个页面都是独立的静态文件，不依赖后端服务，可直接在浏览器中打开或通过静态托管服务访问。

## 目录说明

```
├── index.html          # 仓库首页（可选）
├── pages/              # 存放各独立 HTML 页面
│   ├── example1.html   # 示例页面 1
│   ├── example2.html   # 示例页面 2
│   └── ...
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：核心目录，存放所有独立 HTML 页面文件。
- `assets/`：存放页面共享的样式、脚本或图片资源。
- 根目录的 `index.html` 可作为仓库的导航入口，非必须。

## 页面归档说明

- 所有页面均为纯静态 HTML，可直接双击打开或部署到任意静态服务器。
- 页面之间互不依赖，可单独使用或删除。
- 文件名建议使用英文小写，避免特殊字符，以保持兼容性。

## 维护说明

- 新增页面：在 `pages/` 目录下创建新的 `.html` 文件，并确保其自包含。
- 更新页面：直接编辑对应文件，无需修改其他文件。
- 删除页面：移除对应文件即可，不影响其他页面。
- 资源管理：公共资源放入 `assets/`，私有资源建议内嵌或与页面文件放在同一目录。
